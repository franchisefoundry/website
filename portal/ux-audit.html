Top 8 Priorities — ordered by user impact
1
Add icons to the sidebar navigation
Every top SaaS product (Linear, Vercel, Gusto, Toast) uses icons alongside nav labels. Without them, the sidebar reads like a developer's debug menu, not a polished product. Users scan icons 3× faster than text.
High impact · ~2 hours
2
Replace emoji icons with SVG icons on the admin dashboard
Stat cards currently use 📋 👤 🏢 🎯 🤝 as icons. This is the single biggest signal that the product is self-built. Lucide or Heroicons SVGs with a tinted background pill looks immediately more professional.
High impact · ~1 hour
3
Strengthen page-level typography hierarchy
The PageHeader uses text-xl (20px) for all page titles. Compare to products like Rippling or HubSpot which use 26–32px bold headings. Heavy reliance on text-xs throughout also makes content feel cramped and low-rent.
High impact · ~3 hours (global change)
4
Introduce the gold accent colour as a design system element
brand-gold (#d4a574) is defined in the config but used only sparingly. It's the warmth that differentiates you from generic B2B tools. Use it for primary CTA buttons, active state accents, and premium indicators — it signals quality in a hospitality context.
High impact · ~4 hours
5
Upgrade the franchisee pre-assignment empty state to an onboarding checklist
Currently just a centred paragraph in a card. The best products (Notion, Gusto, Stripe) turn empty states into progress-driving experiences. Show a 4-step checklist: profile complete → consultation booked → match identified → intro arranged. Users always know their next action.
High impact · ~3 hours
6
Add trend indicators to admin dashboard stat cards
Raw counts don't tell a story. A small "+3 this week" indicator next to each stat metric tells admins at a glance whether the pipeline is growing or stalling. Mindbody, HubSpot and Linear all do this well.
Medium impact · ~2 hours
7
Refine the pipeline progress indicator on the journey page
The current implementation is flat coloured bars with tiny 9px labels below. Upgrade to a connected step indicator with dots, a highlighted current step, and a distinct "next step" CTA. This is core UX for a product whose primary value is guiding people through a journey.
Medium impact · ~2 hours
8
Move notification bell to the top of the sidebar, not inline with nav links
The notification bell is currently appended to the bottom of the nav list as if it's a nav item. Standard pattern (used by every product) is a bell icon in the sidebar header or user area, possibly with an unread count badge.
Medium impact · ~30 mins

Navigation & Sidebar

The sidebar is the skeleton of every portal. It's the first thing users see and the interface they interact with most. Right now it's functional but lacks the icon-driven visual language that signals a polished product.

4 issues
!
No nav icons. Text-only navigation feels generic. Icons give users a spatial anchor for each section — critical for a portal with many sections.
!
Notification bell in wrong position. It's appended to the bottom of nav items like a regular link. It belongs near the user avatar or in a top-right utility area.
~
Sidebar background is flat. A very subtle gradient (top slightly lighter) adds depth and signals that this is a premium product, not a default stylesheet.
~
User card at bottom is minimal. The full-name + "Profile & settings" link is good but the avatar area could show a role badge (e.g. "Admin" / "Franchisee") so users always know their context.

Admin Dashboard

The first screen admins see every day. Currently shows counts in emoji-icon cards + two list panels. Good structure, weak visual execution — emoji icons and flat typography undercut the premium positioning.

4 issues
!
Emoji stat icons (📋 👤 🏢 🎯 🤝) are the biggest single signal that this is a self-built tool. Replace with Lucide/Heroicons SVGs in tinted background containers.
!
No trend data on stat cards. A static count of "12 leads" tells you nothing. "+3 this week" or a tiny sparkline tells a story and drives action.
~
PageHeader h1 is text-xl (20px). For a dashboard header, this reads like a section title, not a page. 26–28px bold creates the visual authority this screen needs.
~
Activity feeds use tiny dividers and the same visual weight for every row. Meeting-requested leads should pop visually with a coloured left border or highlight row, not just a small amber badge.

Franchisee Dashboard — Pre-Assignment State

This is what a new franchisee sees before they've been matched. Currently it's a centred paragraph in a card. This is the highest-stakes screen in the product — it needs to drive engagement and reduce drop-off.

3 issues
!
Empty state doesn't guide users. A single centred paragraph gives no clear next action. Users don't know what to do or how long anything takes. This is where drop-off happens.
~
Profile completeness CTA is buried. The "Complete your profile — 65% done →" link appears as small inline text below a paragraph. It should be a primary visual element — this is the most important action on the page.
✓
Stat cards (Brands matched, Interested, Intros) look good but use inconsistent text sizes across the three cards when values are at 0.

Franchisee Journey Page — Pipeline Progress

This is the most emotionally significant screen for franchisees — it shows exactly where they are in their franchise journey. The content logic is strong; the visual execution needs upgrading to match the emotional weight of the moment.

3 issues
!
Pipeline bars are too thin and flat. 8px bars with 9px labels below feel like a loading indicator, not a meaningful journey visualisation. Users should feel clear progress at a glance.
~
"Backup options — in case your primary doesn't progress" is a demotivating frame. Rename to "Alternative paths" or "Additional matches" — warmer language that doesn't imply failure.
✓
The brand hero card and stage guidance sections are well structured — just needs visual elevation (see mock below).

2 issues
~
Cards lack visual hierarchy between name, stage and budget. All text is roughly the same visual weight. A clearer name → stage pill → budget hierarchy would let admins scan faster.
✓
Column headers are clean but the coloured dot + label could be replaced with a left-border accent on the entire column to create stronger visual separation — a pattern used by Linear and Notion databases.

Partner Marketplace

The marketplace is one of the most polished parts of the portal. The blur/lock pattern is smart. Main upgrades are visual — partner cards can feel more premium, and the filter pills could have icons.

2 issues
~
Partner cards use text-only sector badges. Small coloured icons alongside the category label (e.g. a £ icon for Finance) would make scanning faster and feel more considered.
✓
The gold brand colour isn't used at all here. A gold accent strip at the top of each "featured" partner card, or a gold CTA button for unlocked cards, would reinforce the premium brand feel.
